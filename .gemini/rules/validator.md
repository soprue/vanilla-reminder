# Change Validator Rules (검증 에이전트)

이 규칙은 변경 사항이 프로젝트 전체의 안정성을 깨뜨리지 않았는지 확인하기 위해 사용됩니다.

## 1. Fast Check (기본 검증)
다음은 작업을 마칠 때마다 반드시 수행해야 할 기본 단계입니다:
- **타입 정합성**: `npx tsc --noEmit`을 실행하여 모든 파일의 타입 에러를 확인합니다.
- **참조 정합성**: `import/export` 경로가 정확한지, 삭제된 멤버를 참조하는 곳은 없는지 확인합니다.
- **IPC 정합성**: `preload.cjs`에 정의된 API 명칭과 렌더러 프로세스에서 호출하는 명칭이 일치하는지 대조합니다.

## 2. Full Check (선택적 검증)
다음은 주요 기능 추가나 대규모 수정 후에 선택적으로 수행합니다:
- **번들링 확인**: `npm run build` (또는 `webpack`)를 실행하여 번들링 오류 여부를 확인합니다.
- **의존성 순환**: 파일 간의 순환 참조(Circular Dependency)가 발생했는지 검토합니다.

## 3. 리포트 및 수정 제안
검증에 실패한 경우, 다음과 같은 형식으로 보고해야 합니다:
- **[에러 위치]**: 파일명 및 줄 번호.
- **[원인]**: 에러 메시지 또는 논리적 결함 설명.
- **[수정 제안]**: 에러를 해결하기 위해 수정해야 할 코드나 권장 조치.
